# Admissions Domain - Context

**Purpose**: Who this domain serves, top workflows, and terminology.

**Audience**: Product Managers, Designers, Engineers, New Team Members

**Last Updated**: 2025-12-20

---

## Who It Serves

### Primary Users
- **Admissions Officers** - Process leads, manage queue, review applications
- **Prospective Students** - Submit inquiries, receive program recommendations
- **Admissions Leadership** - Monitor metrics, configure workflows

### Secondary Users
- **Support Staff** - Assist with inquiries
- **Analytics Team** - Review performance data

---

## Top Workflows

### 1. Lead Intake and Qualification
1. Prospective student submits inquiry
2. System captures lead data
3. Lead enters qualification queue
4. Agent reviews and qualifies lead
5. Lead moves to appropriate workflow

### 2. Program Matching
1. Student completes program match quiz
2. AI analyzes responses and profile
3. System recommends matching programs
4. Student views recommendations
5. Student expresses interest in programs

### 3. Queue Management
1. Agent opens queue
2. System presents prioritized items
3. Agent reviews item details
4. Agent takes action (approve, reject, request more info)
5. Item moves to next stage

### 4. Segmentation and Outreach
1. Admin creates segment definition
2. System identifies matching contacts
3. Admin configures outreach campaign
4. System sends targeted messages
5. System tracks engagement

---

## Terminology

- **Lead** - Prospective student inquiry or application
- **Program Match** - AI-powered program recommendation system
- **Queue** - Workflow management system for agent review
- **Segment** - Group of contacts sharing common characteristics
- **Workspace** - Organizational boundary (e.g., "admissions", "registrar")
- **App** - Application context (e.g., "student-lifecycle")
- **Mode** - Working mode (operator, leadership, global, workspace)

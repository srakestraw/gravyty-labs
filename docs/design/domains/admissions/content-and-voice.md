# Admissions - Content and Voice

**Purpose**: Tone, messaging rules, and accessibility guidelines.

**Audience**: Designers, Engineers, Content Writers

**Last Updated**: 2025-12-20

---

## Tone and Voice

### Voice Characteristics
- **Welcoming**: Make prospective students feel valued
- **Clear**: Use plain language, avoid jargon
- **Supportive**: Guide students through the process
- **Professional**: Maintain institutional credibility

### Messaging Rules
- Use "you" to address students directly
- Avoid acronyms without explanation
- Use active voice
- Keep sentences concise

---

## Content Patterns

### Form Labels
- Use clear, descriptive labels
- Include help text for complex fields
- Show required field indicators

### Error Messages
- Explain what went wrong
- Suggest how to fix it
- Use friendly, non-blaming language

### Success Messages
- Confirm the action completed
- Explain what happens next
- Provide next steps if applicable

---

## Accessibility Notes

### Alt Text
- Images: Describe content, not decoration
- Icons: Describe function, not appearance
- Charts: Describe data, not visual design

### ARIA Labels
- Icon-only buttons: Describe action
- Status messages: Use `aria-live` regions
- Form errors: Associate with fields

### Screen Reader Support
- Announce state changes
- Provide skip links
- Use semantic HTML
